## 一、线程创建
```
pthread_create(&thread, NULL, my_function, &my_data);
//                  1        2      3            4
```

&thread	工牌：创建完线程后，系统会给你一个"工牌"，以后你靠这个工牌找到这个线程（比如等它干完活、杀掉它）	

身份证号
NULL	特殊要求：比如优先级、栈大小。一般不用管，传NULL表示"默认就行"	不填就默认

my_function	任务内容：这个线程要执行哪个函数	"去搬砖"

&my_data	任务所需的材料：传给那个函数的参数	"搬10块砖"

## 二、等待线程完成
调用函数pthread_join()

并非所有多线程代码都使用join函数。例如，多线程Web服务器可能会创建大量工作线程，然后使用主线程接受请求，并将其无限期地传递给工作线程。因此这样的长期程序可能不需要join。然而，创建线程来（并行）执行特定任务的并行程序，很可能会使用join来确保在退出或进入下一阶段计算之前完成所有这些工作。

## 四、条件变量
```
int shared_data;
bool data_ready = false;
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t cond = PTHREAD_COND_INITIALIZER; // 条件变量

// 消费者线程
void* consumer(void* arg) {
    pthread_mutex_lock(&mutex);
    // 使用 while 循环检查条件（防止虚假唤醒）
    while (!data_ready) {
        // 关键：原子地释放 mutex，并让线程睡眠，等待 cond 信号
        pthread_cond_wait(&cond, &mutex); // <--- 高效睡眠
        // 被唤醒后，会自动重新获取 mutex
    }
    // 条件满足，处理数据
    printf("Consumed: %d\n", shared_data);
    data_ready = false;
    pthread_mutex_unlock(&mutex);
    return NULL;
}

// 生产者线程
void* producer(void* arg) {
    pthread_mutex_lock(&mutex);
    shared_data = 42;
    data_ready = true;
    // 发送信号，唤醒一个（或所有）在 cond 上等待的线程
    pthread_cond_signal(&cond); // <--- 发送信号
    pthread_mutex_unlock(&mutex);
    return NULL;
}
```


## 五、线程API注意点
1. 保持简洁：锁和信号的代码需要保持简单容易
2. 尽量减少线程之间的交互
3. 初始化锁和条件变量
4. 检查返回值
5. 注意传给线程的参数和返回值：比如传递栈上分配的变量引用就是犯错误
6. 每个线程有自己的栈
7. 线程之间总是用条件变量发送信号，切记不要用标记变量同步
