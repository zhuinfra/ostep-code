## 一、基址加界限机制，又称动态重定位(基于硬件)
每个cpu需要两个硬件寄存器：
1. 基址寄存器(Base Register)：存放进程的基址
2. 界限寄存器(Limit Register)：存放进程的界限
进程运行时，所有产生的内存引用都会被处理器通过以下方式转为物理地址
physical_address = base_register + virtual_address

案例
```
void func() {
    int x;
    x = x + 3;
}

128: mov 0x0(%ebx), %eax
132: add $0x03, %eax
135: mov %eax, 0x0(%ebx)
```
PC设置成128,硬件获取指令会将128+基址寄存器中的32KB(32768)，得到32896。
界限寄存器提供访问保护，假设界限寄存器是16KB, 如果进程超过这个界限或者访问负数的虚拟地址则cpu触发异常。
这种基址寄存器配合界限寄存器的硬件结构是芯片中的（每个CPU一对）​。有时我们将CPU的这个负责地址转换的部分统称为内存管理单元（Memory Management Unit，MMU）​。

## 二、操作系统的职责(动态重定位)
### 1.内存管理
1.新进程创建时, 检索空闲列表free list, 为新地址空间标记为已用
2.进程终止，回收内存，放回空闲列表，并且清除相关数据结构。
### 2.基址/界限管理
1.必须在上下文切换时正确保存和回复基址和界限寄存器到内存中，如进程结构或PCB进程控制块。
### 3.异常处理
1.当进程越界访问, cpu触发异常, 操作系统必须终止进程
